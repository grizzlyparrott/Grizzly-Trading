Bookmap Connections and Data Feeds: How to Get Real-Time Order Flow

Bookmap is worthless without a proper data feed. The platform needs real-time depth, liquidity, and executed order data—otherwise the heatmap and bubbles won’t mean anything. This guide shows you how to connect Bookmap to the three feeds that matter: Rithmic, CQG, and dxFeed.

Supported Bookmap Data Feeds

Bookmap supports three main data sources depending on what you’re trading:

FeedUsed ForNotes
RithmicFuturesFastest feed for depth; used by prop firms
CQGFuturesWidely supported; stable but slightly slower
dxFeedCrypto & US equitiesGood for Bookmap’s crypto package

1. Connecting Bookmap to Rithmic

Rithmic is the fastest and most detailed futures feed available. If you’re using a prop firm like Apex or Topstep, you’re almost certainly connecting through Rithmic.

Steps:

  1. Open Bookmap
  2. Go to Connections → Configure
  3. Click Add → choose Rithmic
  4. Select your login type (Live, Paper, or Prop Firm)
  5. Enter username/password
  6. Choose the correct server location (faster if closer)
  7. Enable Market Depth and Full Order Book
  8. Click Connect

If you’re with a prop firm, make sure you use their credentials—not your broker credentials. They are not the same.

Common Rithmic Problems

  • Incorrect server selection → no connection
  • Using old login credentials from expired evals
  • Trying to connect during maintenance windows

Rithmic is fast but picky. If one thing is off, the connection fails.

2. Connecting Bookmap to CQG

CQG is the most widely supported futures data feed, especially for retail brokers.

Steps:

  1. Go to Connections → Configure
  2. Click Add → CQG
  3. Enter your CQG username & password
  4. Enable Futures data
  5. Turn on Full Depth of Market if available
  6. Connect

Some brokers restrict depth unless you specifically subscribe, so check your market data package.

3. Connecting Bookmap to dxFeed

dxFeed is used for:

  • Crypto order flow
  • US equities
  • Some global markets

Steps:

  1. Go to Connections → Configure
  2. Select dxFeed
  3. Sign in with your Bookmap email/password
  4. Select the markets you subscribed to
  5. Connect

dxFeed is the easiest feed to connect since Bookmap manages the subscription directly.

Which Feed Should You Use?

If You Trade…Use This
Futures through a prop firmRithmic
Futures through a brokerCQG or Rithmic
CryptodxFeed
US equitiesdxFeed

If you only care about futures, Rithmic is the cleanest and fastest feed.

Required Subscriptions

  • Bookmap subscription (Global or above)
  • Market data fees for each exchange
  • Feed-specific fees depending on provider

Bookmap is not plug-and-play—you must have the correct exchange data activated or nothing will load.

Troubleshooting Common Problems

1. Bookmap loads but shows no liquidity

You subscribed to top-of-book only. You need full depth.

2. Error: “Login failed”

  • Wrong server (Rithmic)
  • Expired credentials (prop firm)
  • Wrong username format (CQG)

3. Heatmap shows delayed movement

  • You’re using demo data
  • Your feed is throttled
  • You’re on a VPN

4. Depth missing on CQG

You didn’t subscribe to the depth package. Most brokers disable it by default.

Final Thoughts

Bookmap only works if you feed it clean, full-depth market data. Rithmic, CQG, and dxFeed all get the job done, but the right choice depends on what you’re trading. Set it up correctly once, save your connection profile, and your order flow will be clean every session.
